## 3. 数据库设计

### 3.1 实体关系图 (ER图)

```
+-----------------+       +-------------------+       +-------------------+
|     users       |       |  health_records   |       |  chat_records     |
+-----------------+       +-------------------+       +-------------------+
| id (PK)         |<----->| user_id (FK)      |<----->| user_id (FK)      |
| username        |       | systolic          |       | message           |
| email           |       | diastolic         |       | role              |
| password_hash   |       | blood_sugar       |       | created_at        |
| created_at      |       | measured_at       |       +-------------------+
| updated_at      |       | created_at        |
+-----------------+       +-------------------+
```

### 3.2 表结构设计

#### 3.2.1 用户表 (users)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | NOT NULL | 用户名 |
| email | VARCHAR(100) | UNIQUE, NOT NULL | 邮箱 |
| password_hash | VARCHAR(255) | NOT NULL | 密码哈希 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

#### 3.2.2 健康数据表 (health_records)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 记录ID |
| user_id | INT | FOREIGN KEY(users.id) | 用户ID |
| systolic | INT | NOT NULL | 收缩压（高压） |
| diastolic | INT | NOT NULL | 舒张压（低压） |
| blood_sugar | FLOAT | NOT NULL | 血糖值 |
| measured_at | DATETIME | NOT NULL | 测量时间 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

#### 3.2.3 聊天记录表 (chat_records)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 记录ID |
| user_id | INT | FOREIGN KEY(users.id) | 用户ID |
| message | TEXT | NOT NULL | 消息内容 |
| role | ENUM('user', 'ai') | NOT NULL | 角色 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

## 4. API接口设计

### 4.1 认证相关接口

- **路径**: `/api/auth/register`
- **方法**: `POST`
- **请求体**:
  ```json
  {

    "email": "zhangsan@example.com",
    "password": "Password123!",
    "confirm_password": "Password123!"
  }

- **响应**:

  {

      "email": "zhangsan@example.com"
    }
  }
  ```

### 4.2 健康数据相关接口

#### 4.2.1 提交血压血糖数据接口

- **路径**: `/api/health/records`
- **方法**: `POST`
- **请求头**: `Authorization: Bearer <token>`
- **请求体**:
  ```json
  {
    "systolic": 120,
    "diastolic": 80,
    "blood_sugar": 5.6,
    "measured_at": "2026-04-18T08:30:00"
  }
  ```
- **响应**:
  ```json
  {
    "id": 1,
    "user_id": 1,
    "systolic": 120,
    "diastolic": 80,
    "blood_sugar": 5.6,
    "measured_at": "2026-04-18T08:30:00",
    "created_at": "2026-04-18T08:30:00"
  }
  ```

#### 4.2.2 获取历史数据接口

- **路径**: `/api/health/records`
- **方法**: `GET`
- **请求头**: `Authorization: Bearer <token>`
- **查询参数**:
  - `start_date`: 开始日期 (可选)
  - `end_date`: 结束日期 (可选)
  - `limit`: 限制数量 (可选，默认20)
  - `offset`: 偏移量 (可选，默认0)
- **响应**:
  ```json
  {
    "total": 10,
    "records": [
      {
        "id": 1,
        "systolic": 120,
        "diastolic": 80,
        "blood_sugar": 5.6,
        "measured_at": "2026-04-18T08:30:00"
      }
    ]
  }
  ```

#### 4.2.3 获取数据分析接口

- **路径**: `/api/health/analysis`
- **方法**: `GET`
- **请求头**: `Authorization: Bearer <token>`
- **查询参数**:
  - `period`: 周期 (可选，默认"week"，可选值："day", "week", "month", "year")
- **响应**:
  ```json
  {
    "average": {
      "systolic": 125,
      "diastolic": 82,
      "blood_sugar": 5.8
    },
    "trend": "stable",
    "abnormal_count": 2,
    "suggestions": [
      "血压略高，建议减少盐分摄入",
      "血糖在正常范围内，继续保持"
    ]
  }
  ```

## 6. 配置管理

### 6.1 环境变量（统一从 `.env` 读取）

**服务基础**
- `APP_NAME` / `APP_HOST` / `APP_PORT`（默认 8001）/ `APP_ENV` / `APP_DEBUG`

**数据库**
- `DATABASE_URL`：MySQL 连接字符串（**必填**；`config/settings.py` 默认 `None`，启动时无此变量会 `RuntimeError`）

**JWT**
- `SECRET_KEY` / `ALGORITHM`（默认 `HS256`）/ `ACCESS_TOKEN_EXPIRE_MINUTES`（默认 1440）

**LLM**
- `LLM_API_KEY` / `LLM_MODEL`（默认 `qwen3.6-plus`）/ `LLM_BASE_URL`（默认通义千问兼容模式）

**嵌入（DashScope 云端）**
- `EMBEDDING_MODEL`（默认 `text-embedding-v3`）/ `DASHSCOPE_API_KEY` / `EMBEDDING_DIM`（默认 1024）

**Milvus**
- `MILVUS_URI` / `MILVUS_DB`（默认 `medical_RAG`）/ `MILVUS_COLLECTION`（默认 `medical_knowledge`）/ `MILVUS_TOKEN`

**文档切分**
- `CHUNK_SIZE`（默认 500）/ `CHUNK_OVERLAP`（默认 50）

**SMTP 邮件**
- `SMTP_SERVER`（默认 `smtp.qq.com`）/ `SMTP_PORT`（默认 587）/ `SMTP_SENDER_EMAIL` / `SMTP_USERNAME` / `SMTP_PASSWORD`

**MCP**
- `MCP_SERVER_URL`（默认 `http://localhost:8888`）

### 6.2 配置文件结构

`config/settings.py` 用 `pydantic-settings.BaseSettings` 加载上述变量，`.env` 是真源，源码中无明文密钥：

```python
# config/settings.py（精简示例，省略中间字段）
from pydantic_settings import BaseSettings
from typing import Optional

class Settings(BaseSettings):
    APP_PORT: int = 8000
    DATABASE_URL: Optional[str] = None  # 必填，启动时校验
    SECRET_KEY: str = "change-me-in-production"
    LLM_API_KEY: Optional[str] = None
    LLM_MODEL: str = "qwen3.6-plus"
    LLM_BASE_URL: str = "https://dashscope.aliyuncs.com/compatible-mode/v1"
    DASHSCOPE_API_KEY: Optional[str] = None
    MILVUS_URI: str = "http://localhost:19530"
    # ... 其他 SMTP / MCP 字段省略

    class Config:
        env_file = ".env"
        env_file_encoding = "utf-8"
        case_sensitive = False

    def __init__(self, **values):
        super().__init__(**values)
        if not self.DATABASE_URL:
            raise RuntimeError(
                "DATABASE_URL 未配置：请在 AI_disease_backend/.env 中设置 DATABASE_URL"
            )

settings = Settings()
```

## 7. 部署方案

### 7.1 本地开发

1. 安装依赖: `pip install -r requirements.txt`
2. 启动数据库: 确保 MySQL 服务运行（首次执行 `mysql -u root -p < diease.sql`）
3. 运行迁移（如需管理表结构变更；`pyproject.toml` 已不含 `db_url`，必须 CLI 显式传）：
   ```bash
   export $(grep -v '^#' AI_disease_backend/.env | xargs)
   aerich upgrade --db-url "$DATABASE_URL"
   ```
4. 启动应用: `uvicorn main:app --reload`（端口由 `.env` 的 `APP_PORT` 控制）

### 7.2 生产部署

1. 使用 Docker 容器化
2. 配置 Nginx 作为反向代理
3. 使用 PM2 或 Supervisor 管理进程
4. 配置 HTTPS
5. 定期备份数据库

## 8. 安全措施

1. **密码加密**: 使用 PBKDF2（`pbkdf2_sha256`）对密码进行哈希处理
2. **JWT 认证**: 使用 JWT 进行无状态认证
3. **CORS 配置**: 合理配置 CORS 策略
4. **输入验证**: 使用 Pydantic 进行数据验证
5. **SQL 注入防护**: 使用 ORM 和参数化查询
6. **敏感数据保护**: 避免在响应中返回敏感信息
7. **日志记录**: 记录关键操作和异常
8. **配置隔离**: 全部密钥通过 `.env` 注入，源码中无明文；`.env` 由 `.gitignore` 排除

## 9. 性能优化

1. **数据库索引**: 为频繁查询的字段创建索引
2. **缓存**: 使用 Redis 缓存热点数据
3. **异步处理**: 使用 FastAPI 的异步特性
4. **批量操作**: 减少数据库查询次数
5. **负载均衡**: 部署多个实例进行负载均衡
