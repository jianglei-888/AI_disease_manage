# 慢性病管理微信小程序需求分析

## 1. 产品概述

### 1.1 产品背景

随着慢性病（如高血压、糖尿病）患者数量的增加，日常健康监测和管理变得尤为重要。传统的健康管理方式存在数据记录不及时、分析不专业等问题。本项目旨在开发一款基于微信小程序的慢性病管理系统，通过智能监测和 AI 辅助，帮助用户更好地管理血压和血糖等健康指标。

### 1.2 产品目标

- 提供便捷的血压、血糖数据记录功能
- 通过 AI 技术为用户提供个性化的健康建议
- 建立用户健康档案，方便长期跟踪管理
- 提高用户的健康意识和自我管理能力

### 1.3 目标用户

- 高血压、糖尿病等慢性病患者
- 需要定期监测血压血糖的中老年人
- 关注健康管理的普通人群

## 2. 功能需求

### 2.1 微信小程序端功能

#### 2.1.1 注册功能
- 用户输入邮箱、用户名、密码和确认密码
- 密码强度验证
- 邮箱格式验证
- 注册成功后自动登录

#### 2.1.2 登录功能
- 输入邮箱和密码
- 记住密码选项
- 忘记密码功能
- 登录状态保持

#### 2.1.3 首页功能
- 显示今日血压血糖卡片，展示最新记录的数据
- 显示饮食建议卡片
- 显示运动建议卡片
- 快速添加血压血糖记录的入口
- 进入聊天功能的入口

#### 2.1.4 提交血压血糖功能
- 提供输入框，供用户输入高压、低压和血糖值
- 记录测量时间
- 提交后显示数据趋势图表
- 异常值提醒

#### 2.1.5 聊天功能
- 与 AI 进行交互，获取用药咨询
- 提供饮食建议
- 健康知识问答
- 聊天历史记录

#### 2.1.6 个人中心功能
- 退出登录
- 用户信息显示（头像、用户名、邮箱）
- 免责声明
- 隐私政策
- 关于我们

### 2.2 后端 API 功能

#### 2.2.1 用户管理
- 注册接口
- 登录接口
- 获取用户信息接口
- 修改用户信息接口

#### 2.2.2 健康数据管理
- 提交血压血糖数据接口
- 获取历史数据接口
- 获取数据分析接口

#### 2.2.3 AI 服务
- 用药咨询接口
- 饮食建议接口
- 健康知识问答接口

## 3. 技术架构

### 3.1 系统架构
- **前端**：微信小程序原生开发
- **后端**：FastAPI + MySQL 8
- **AI 服务**：LangChain + LangGraph
- **向量数据库**：Milvus（药品 / 用药 / 指南检索）
- **工具调用**：FastMCP

### 3.2 技术栈选择

| 技术 | 版本 | 用途 |
|------|------|------|
| 微信小程序 | 最新 | 前端用户界面 |
| FastAPI | 最新 | 后端 API 服务 |
| MySQL | 8.0+ | 数据存储 |
| LangChain | 1.2 | AI 应用框架 |
| LangGraph | 1.1 | 复杂 AI 工作流 |
| Milvus | 2.6+ | 向量数据库 |
| DashScope | 最新 | 通义千问 LLM + text-embedding-v3 嵌入（云端 API） |
| FastMCP | 最新 | 工具调用平台 |

## 4. 数据结构

### 4.1 用户表（users）
| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | NOT NULL | 用户名 |
| email | VARCHAR(100) | UNIQUE, NOT NULL | 邮箱 |
| password_hash | VARCHAR(255) | NOT NULL | 密码哈希 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

### 4.2 健康数据表（health_records）
| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 记录ID |
| user_id | INT | FOREIGN KEY(users.id) | 用户ID |
| systolic | INT | NOT NULL | 收缩压（高压） |
| diastolic | INT | NOT NULL | 舒张压（低压） |
| blood_sugar | FLOAT | NOT NULL | 血糖值 |
| measured_at | DATETIME | NOT NULL | 测量时间 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 4.3 聊天记录表（chat_records）
| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 记录ID |
| user_id | INT | FOREIGN KEY(users.id) | 用户ID |
| message | TEXT | NOT NULL | 消息内容 |
| role | ENUM('user', 'ai') | NOT NULL | 角色 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

## 6. 安全要求

### 6.1 数据安全
- 密码加密存储（PBKDF2）
- 敏感数据传输加密（HTTPS / JWT）
- 定期数据备份

### 6.2 访问控制
- 用户身份验证（JWT）
- 权限管理
- 防止 SQL 注入攻击（ORM + 参数化查询）

### 6.3 隐私保护
- 遵守相关隐私法规
- 用户数据隔离（按 user_id 过滤）
- 明确的隐私政策

### 6.4 配置隔离
- 全部密钥通过 `.env` 注入
- 源码中无明文密钥
- `.env` 由 `.gitignore` 排除，避免误提交

## 7. 部署方案

### 7.1 前端部署
- 微信小程序审核发布
- 版本管理

### 7.2 后端部署
- Docker Compose 一键启动（MySQL + Milvus + backend）
- 服务器配置
- 数据库部署
- API 服务部署

### 7.3 AI 服务部署
- LangChain 和 LangGraph 配置
- Milvus 向量数据库部署（独立 / 远程均可，地址由 `.env` 的 `MILVUS_URI` 控制）
- 嵌入走 DashScope 云端 API，无需本地服务
- FastMCP 工具服务（可选独立进程）
